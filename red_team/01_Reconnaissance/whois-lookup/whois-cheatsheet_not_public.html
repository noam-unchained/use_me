WHOIS Lookup — cheat sheet

Pulls WHOIS registration data for one or more domains — registrar, dates, nameservers, and registrant email.

Step 1 — Install & Run
pip install -r requirements.txt
python3 whois_lookup.py example.com
Step 2 — Usage
# Single domain
python3 whois_lookup.py example.com

# Multiple domains
python3 whois_lookup.py example.com target.org

# Save to JSON
python3 whois_lookup.py example.com target.org -o results.json
Step 3 — What It Returns
FieldWhat it tells you
RegistrarWho the domain is registered with
Creation dateWhen the domain was first registered (age = trust signal)
Expiration dateWhen it expires — expiring soon = acquisition opportunity
Name serversHosting / DNS provider — reveals infrastructure
Registrant emailContact email — useful for OSINT on the org
CountryRegistrant country
DNSSECWhether the domain has DNSSEC enabled
Reading the Output
Domain     : example.com
Registrar  : GoDaddy
Created    : 1995-08-14
Expires    : 2025-08-13
Nameservers: ns1.example.com, ns2.example.com
Email      : admin@example.com
Country    : US
DNSSEC     : unsigned

A very young domain (days or weeks old) combined with suspicious content is a strong phishing indicator.

Key idea: domain age is one of the strongest signals. Phishing domains are almost always newly registered (under 30 days). Combine with the phishing-detector for URL risk scoring.
