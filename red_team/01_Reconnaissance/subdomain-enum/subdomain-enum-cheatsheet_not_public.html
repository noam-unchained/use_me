Subdomain Enumeration — cheat sheet

Automates subdomain discovery by running Subfinder, Assetfinder, and Sublist3r in sequence, then merging all results.

Step 1 — Install the Three Tools
# Subfinder
go install -v github.com/projectdiscovery/subfinder/v2/cmd/subfinder@latest

# Assetfinder
go install github.com/tomnomnom/assetfinder@latest

# Sublist3r
pip install sublist3r
Step 2 — Run
chmod +x subdomain_enum.sh
./subdomain_enum.sh example.com
# also accepts full URLs:
./subdomain_enum.sh https://example.com/anything
Step 3–5 — Three Tools, Different Sources
ToolData SourcesBest for
SubfinderCert transparency, Shodan, VirusTotal, APIsPassive, fast, comprehensive
AssetfinderDNS records, web crawl, related assetsFinding related domains
Sublist3rGoogle, Bing, Yahoo, DNSdumpster, crt.shSearch engine enumeration
Step 6 — Merge & Deduplicate

All three tool outputs are combined and deduplicated automatically with sort -u. You get one clean file with no duplicates.

Step 7 — Reading the Output
[+] Running Subfinder...      Found 43 subdomains
[+] Running Assetfinder...    Found 31 subdomains
[+] Running Sublist3r...      Found 27 subdomains

[*] Total unique subdomains:  61
[*] Results saved to: subdomains_example.com.txt

Open the results file and pipe into other tools:

cat subdomains_example.com.txt | httpx      # find live hosts
cat subdomains_example.com.txt | nmap ...  # port scan them

Key idea: no single tool finds everything — running all three and merging gives the best coverage. Subfinder usually finds the most, Assetfinder catches related assets the others miss.
